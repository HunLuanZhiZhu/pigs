## 方案 B 实施计划：OutputSink 统一命令输出通道

### 设计原理

给 `Agent` 增加一个 `output: OutputSink` 字段，所有命令输出（`commands.rs`、`models.rs`、`doctor.rs` 中的 `println!`/`eprintln!`）改为调用 `agent.output().println(...)`。两种 sink 实现：

- `OutputSink::Stdout` —— REPL 模式用，直接 `println!` 到 stdout（保留现状行为）
- `OutputSink::Buffer` —— TUI 模式用，累积到 `String`，调用方在 `handle_command` 返回后取走并 `app.push_to_main_chat(ChatEntry::System(...))`

**不遮蔽 stdout**，也不用改 `handle_command` 签名。

### 实施步骤

#### 步骤 1：新增 `OutputSink` 类型（新文件 `crates/pigs-cli/src/output.rs`）

```rust
use std::fmt::Display;

/// Command output channel — abstracts over stdout (REPL) vs in-memory buffer (TUI).
pub enum OutputSink {
    /// Plain stdout (REPL mode).
    Stdout,
    /// In-memory buffer collected by TUI caller.
    Buffer(String),
}

impl Default for OutputSink {
    fn default() -> Self { Self::Stdout }
}

impl OutputSink {
    pub fn println(&mut self, s: impl Display) {
        match self {
            OutputSink::Stdout => println!("{s}"),
            OutputSink::Buffer(buf) => {
                buf.push_str(&s.to_string());
                buf.push('\n');
            }
        }
    }

    pub fn println_empty(&mut self) {
        match self {
            OutputSink::Stdout => println!(),
            OutputSink::Buffer(buf) => buf.push('\n'),
        }
    }

    pub fn eprintln(&mut self, s: impl Display) {
        // Errors also go through the same channel (TUI shouldn't bypass sink)
        self.println(s);
    }

    /// Take the accumulated buffer (TUI mode), clearing it.
    /// Returns empty string for Stdout mode.
    pub fn take_buffer(&mut self) -> String {
        match self {
            OutputSink::Stdout => String::new(),
            OutputSink::Buffer(buf) => std::mem::take(buf),
        }
    }

    pub fn is_buffer(&self) -> bool {
        matches!(self, OutputSink::Buffer(_))
    }
}
```

#### 步骤 2：在 `Agent` 上加字段和方法（`crates/pigs-cli/src/agent.rs`）

- 在 `pub struct Agent` 字段中 `output_format` 之后加 `pub output: OutputSink`
- 在所有 `Ok(Self { ... })` 构造点初始化 `output: OutputSink::default()`
- 加便捷方法：
  ```rust
  pub fn output(&mut self) -> &mut OutputSink { &mut self.output }
  pub fn set_output_buffer_mode(&mut self) { self.output = OutputSink::Buffer(String::new()); }
  pub fn take_output_buffer(&mut self) -> String { self.output.take_buffer() }
  ```

#### 步骤 3：改造 `crates/pigs-cli/src/commands.rs`

- `handle_command` 入口先取一个本地引用：`let out = &mut agent.output;`
- 把 162 处 `println!(...)` 改为 `out.println(...)`，30 处 `eprintln!(...)` 改为 `out.eprintln(...)`
- 空行 `println!()` 改为 `out.println_empty()`
- `print_help(lang)` 和 `print_mcp_help()` 改为接收 `&mut OutputSink` 参数：`fn print_help(out: &mut OutputSink, lang: Language)`
- 调用 `crate::models::print_models(...)` 等外部函数时传入 `out`

#### 步骤 4：改造 `crates/pigs-cli/src/models.rs`

- `print_models(config, current_selection, _current_remote)` → `print_models(out: &mut OutputSink, config, current_selection, _current_remote)`
- `print_model_status(agent)` → 内部用 `agent.output.println(...)`
- `handle_model_command(agent, arg)` → 内部用 `agent.output.println(...)` / `agent.output.eprintln(...)`
- `run_model_add_wizard(agent)` → 同上

#### 步骤 5：改造 `crates/pigs-cli/src/doctor.rs`

- `print_doctor_report(items)` → `print_doctor_report(out: &mut OutputSink, items)`
- `run_doctor(agent)` → 内部用 `agent.output.println(...)`

#### 步骤 6：改造 TUI 调用方（`crates/pigs-cli/src/tui_repl.rs`）

在 `handle_command` 调用前后加缓冲切换逻辑（仅对 `/` 开头的命令）：

```rust
if text.starts_with('/') {
    // Switch to buffer mode and capture output
    agent.set_output_buffer_mode();
    let result = handle_command(agent, &text).await;
    let captured = agent.take_output_buffer();
    
    // Push captured output to TUI chat (only if non-empty)
    if !captured.trim().is_empty() {
        app.push_to_main_chat(pigs_tui::chat::ChatEntry::System(captured.trim_end().to_string()));
    }
    
    match result {
        Ok(CommandResult::Continue) => { /* existing nav sync logic */ }
        Ok(CommandResult::Quit) => { /* ... */ }
        Err(e) => {
            let _ = broker.sender().send(AppEvent::AgentError(e.to_string()));
        }
    }
}
```

REPL 调用方（`repl.rs`）**不用改**，`OutputSink::default()` 就是 `Stdout`，行为与现状一致。

#### 步骤 7：修复 `ChatEntry::System` 多行渲染（`crates/pigs-tui/src/chat.rs`）

当前 `ChatEntry::System(text)` 只渲染单行，多行字符串会乱。改为按 `text.lines()` 分行渲染：

```rust
ChatEntry::System(text) => {
    for line in text.lines() {
        lines.push(Line::from(vec![
            Span::styled(format!(" {line}"), Style::default().fg(Color::Yellow)),
        ]));
    }
    lines.push(Line::raw(""));
}
```

#### 步骤 8：处理 `tui_repl.rs` 中的 bash 模式（`!` 前缀）

当前 `run_bash_and_inject` 用 `AppEvent::AgentError` 注入 bash 输出（带 `Error:` 前缀，语义不对）。既然已有 `OutputSink`，可以保持原状（bash 不是本次修复目标），或顺便改为 `ChatEntry::Bash` 变体——但为控制改动范围，**本次不改 bash 模式**。

### 风险与边界

- **`out.println` 借用问题**：`handle_command` 入口 `let out = &mut agent.output;` 会与 `agent.session` 等字段借用冲突。解决：在每次需要输出时单独取 `agent.output.println(...)`，或用 `agent.output_mut()` 返回引用。实际实施时优先用 `agent.output.println(...)` 直接调用（避免长生命周期借用）。
- **外部 helper 函数签名变更**：`print_models`、`print_doctor_report` 等函数签名变化会影响其他调用点。需 grep 确认所有调用点都同步更新。
- **多行渲染**：`ChatEntry::System` 改为分行渲染后，原有单行调用（如 `app.rs:291` 的 `Error: {e}`）不受影响。
- **bash 模式**：不在本次修复范围，保留 `AppEvent::AgentError` 路径（带 `Error:` 前缀，对 bash 输出语义不完美但可用）。

### 不做的事

- 不改 `handle_command` 签名
- 不遮蔽 stdout
- 不改 bash 模式（`!` 前缀）
- 不改 `AppEvent` 枚举（无需新增变体，TUI 同步直接用 `push_to_main_chat`）
- 不改 REPL 调用方（默认 Stdout 兼容）
