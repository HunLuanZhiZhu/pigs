# API 层摘要式上下文压缩实施计划

## 架构设计

### 核心流程

```
请求进入 pigs-proxy 路由层 (server.rs handle())
  ↓
判断 1: 模型名带 -pig?
  是 → 脱帽, 分发给 handle_pig (相位编排)
       编排层通过 loopback 重入路由层（此时是真实模型名）
  否 → 判断 2: 估算 token > context_window × 系数(0.9)?
         超限 → 压缩流程:
           查前缀缓存 → 有缓存: 用缓存替换对应消息 → 重新 loopback 发路由层
                        无缓存: 调 LLM 生成摘要 → 替换 → 存缓存 → 重新 loopback 发路由层
           (循环直到不超限)
         不超限 → 透传上游
```

### 压缩位置

在 `handle_passthrough()` 中，`pick_endpoint()` 之后、`retry::dispatch()` 之前。这是所有上游请求的唯一 chokepoint（外部请求 + 编排 loopback 都经过这里）。

### 压缩模块位置

新建 `crates/pigs-proxy/src/compaction.rs`——proxy 内部模块。

## 实施步骤

### 第 1 步：Endpoint 添加 context_window 配置

**文件**: `crates/pigs-proxy/src/config.rs`

`Endpoint` 和 `EndpointRaw` 新增 `context_window: Option<u64>` 字段。

**文件**: `config.toml` 示例配置更新。

如果没有配置，使用 `default_context_window_for(model)` 默认值（复用 pigs-config 的逻辑：claude* → 200_000，其他 → 128_000）。

### 第 2 步：Token 估算

**文件**: `crates/pigs-proxy/src/compaction.rs`（新建）

```rust
/// Estimate token count for a JSON value (messages array).
/// Uses chars/4 heuristic for now. Counts message text + tool_use input + tool_result output.
fn estimate_tokens(body: &serde_json::Value) -> u64
```

解析 body 中的 `messages` 数组，按协议格式（Chat/Messages/Responses）提取文本内容，估算 token 数。

### 第 3 步：前缀缓存

**文件**: `crates/pigs-proxy/src/compaction.rs`

```rust
/// A compaction cache entry: the compressed version of a message prefix.
struct CompactionCacheEntry {
    /// Hash of the original message prefix that was compacted
    prefix_hash: String,
    /// The summary message (replaces the original prefix)
    summary: serde_json::Value,
    /// How many original messages were compacted into this summary
    compacted_count: usize,
}

/// In-memory compaction cache, keyed by (model, session_conversation_id or prefix_hash)
struct CompactionCache {
    entries: std::collections::HashMap<String, CompactionCacheEntry>,
}
```

前缀哈希：对消息数组的前 N 条消息做 SHA256 哈希（用 `blake3` 或简单字符串拼接哈希）。如果前缀不变，缓存命中。

缓存存储在 `AppState` 中（`Arc<std::sync::Mutex<CompactionCache>>`），进程级内存缓存，重启失效。

### 第 4 步：摘要式压缩（调 LLM）

**文件**: `crates/pigs-proxy/src/compaction.rs`

```rust
/// Compact messages by sending old messages to the LLM for summarization.
/// 1. Split messages: keep last N recent, summarize the rest
/// 2. Build a summarization prompt (structured: Primary Request, Key Files, Current Work, etc.)
/// 3. Send to the SAME upstream endpoint (via loopback, as a regular request)
/// 4. Use the LLM's response as the summary message
/// 5. Replace old messages with [summary_system_message, ...recent_messages]
async fn compact_with_llm(
    body: &serde_json::Value,
    endpoint: &Endpoint,
    protocol: Protocol,
    keep_recent: usize,
    coefficient: f64,
    context_window: u64,
    upstream_client: &UpstreamClient,
) -> Result<serde_json::Value, CompactionError>
```

**摘要请求**：向同一个 upstream endpoint 发一个新请求，body 中 `messages` 包含：
- 一条 system 消息：摘要指令（参考 opencode/cline 的结构化模板：Goal / State / Files / Next）
- 用户消息：序列化的旧对话内容

摘要请求的 `max_tokens` 限制为 4096（参考 opencode）。

**摘要请求也走 loopback**：通过 `LoopbackPhaseTransport` 或直接构造 HTTP 请求发回 `http://127.0.0.1:{port}`，带内部令牌。这样摘要请求本身也会经过路由层（可能触发自身压缩，但摘要请求很小不会超限）。

### 第 5 步：压缩流程编排

**文件**: `crates/pigs-proxy/src/compaction.rs`

```rust
/// Main compaction entry point. Called from handle_passthrough before dispatch.
/// Returns the (possibly compacted) body to send upstream.
pub async fn ensure_context_fits(
    body: serde_json::Value,
    endpoint: &Endpoint,
    protocol: Protocol,
    context_window: u64,
    coefficient: f64,
    cache: &Arc<std::sync::Mutex<CompactionCache>>,
    loopback: &LoopbackConfig,  // base_url + token
) -> Result<serde_json::Value, CompactionError> {
    let estimated = estimate_tokens(&body);
    let threshold = (context_window as f64 * coefficient) as u64;
    
    if estimated <= threshold {
        return Ok(body);  // No compaction needed
    }
    
    // Check cache
    let prefix_hash = hash_prefix(&body);
    if let Some(cached) = cache.lock()...get(&prefix_hash) {
        // Replace prefix with cached summary
        let compacted = replace_prefix_with_summary(&body, &cached)?;
        // Re-submit via loopback (recursive check)
        return ensure_context_fits(compacted, endpoint, protocol, ...).await;
    }
    
    // No cache: do LLM summarization
    let compacted = compact_with_llm(&body, endpoint, protocol, ...).await?;
    // Store in cache
    cache.lock()...insert(prefix_hash, CompactionCacheEntry { ... });
    // Re-submit via loopback
    ensure_context_fits(compacted, ...).await
}
```

**防止无限递归**：设置最大压缩次数（如 5 次），超过则强制截断（fallback 到简单截断）。

### 第 6 步：接入 handle_passthrough

**文件**: `crates/pigs-proxy/src/server.rs`

在 `handle_passthrough()` 中，`pick_endpoint()` 之后、body 清洗之后、`retry::dispatch()` 之前：

```rust
// Context window compaction check
let context_window = endpoint.context_window
    .unwrap_or_else(|| default_context_window_for(&client_model));
let body = compaction::ensure_context_fits(
    body,
    &endpoint,
    protocol,
    context_window,
    state.compaction_coefficient,  // 0.9
    &state.compaction_cache,
    &state.loopback_config,
).await?;
```

### 第 7 步：AppState 添加压缩配置

**文件**: `crates/pigs-proxy/src/server.rs` / `lib.rs`

`AppState` 新增：
- `compaction_cache: Arc<std::sync::Mutex<CompactionCache>>`
- `compaction_coefficient: f64`（默认 0.9，可从 config.toml 配置）
- `loopback_config: LoopbackConfig`（base_url + token，复用已有值）

### 第 8 步：配置

**文件**: `config.toml` 和 `crates/pigs-proxy/src/config.rs`

```toml
[compaction]
enabled = true           # 默认 true
coefficient = 0.9        # 触发阈值系数
keep_recent = 4          # 保留最近 N 条消息
max_rounds = 5           # 最大压缩轮次
summary_max_tokens = 4096 # 摘要请求的 max_tokens
```

`Endpoint` 级别也可以覆盖 `context_window`。

### 第 9 步：摘要提示词

**文件**: `crates/pigs-proxy/src/compaction.rs`（内联常量）

参考 opencode 的结构化模板：
```
You are a conversation summarizer. Summarize the following conversation context.
Output a structured summary with these sections:

## Objective
What the user is trying to accomplish.

## Important Details
Key technical decisions, constraints, and context.

## Work State
- Completed: What has been done.
- Active: What is being worked on.
- Blocked: Any blockers.

## Relevant Files
Files that have been read, modified, or discussed.

## Next Step
What should happen next.

Preserve exact file paths, symbol names, and commands. Never mention the compaction process.
```
