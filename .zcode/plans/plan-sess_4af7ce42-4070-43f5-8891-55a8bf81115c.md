# 统一 CLI LLM 请求路径 + 临时禁用子 Agent 工具

## 背景

当前架构中存在**两条不相通的 LLM 请求路径**：
- CLI 非-pig → `pigs-llm` 直连上游，**完全跳过 pigs-api 和 pigs-proxy**
- CLI pig → 自建 `PhasedRuntime`（用 `pigs-llm` 直连），**跳过 pigs-proxy 的 retry 逻辑**
- 进程中存在两个 `PhasedRuntime` 实例（proxy 的一个用 `ProxyApiClient`，CLI 的一个用 `pigs-llm`）

## 目标架构

```
CLI Agent
  │
  └→ pigs-api PhasedRuntime (统一入口)
        │
        ├─ 非-pig → 单次 LLM 调用（Pre 直接输出 PIGEND，无三阶段编排）
        │            └→ self.api (ProxyApiClient) → dispatch_in_process → 上游
        │
        └─ -pig    → 完整三阶段编排 (Pre→Executor→Post)
                     └→ self.api (ProxyApiClient) → dispatch_in_process → 上游
```

核心变化：**CLI 的 `api_client` 从 `pigs-llm` 直连客户端替换为 `ProxyApiClient`**，所有 LLM 请求统一经过 pigs-proxy 的 retry 逻辑。非-pig 模型通过 PhasedRuntime 走"简单路径"（Pre 阶段直接返回结果，不做编排）。

### 2. PhasedRuntime 新增"直发"路径

**文件**: `crates/pigs-api/src/phased_runtime.rs`

在 `run_turn_with_progress` (line 218) 的消息拆分之后、相位循环之前，新增分支：

```rust
// 非-pig 模式：单次 LLM 调用，不做相位编排
// Non-pig mode: single LLM call, no phased orchestration
if !self.is_pig {
    events.push(ev("phase_start", Some("direct"), None));
    emit(TurnProgress::PhaseStart { phase: "direct".into() });
    let payload = user_question.clone();
    let text = self.run_phase(
        Phase::Pre,   // 复用 Pre 阶段基础设施，但不注入相位提示词
        &base_history,
        &payload,
        &system_prompt,
        progress.as_ref(),
    ).await?;
    let final_text = strip_markers(&text);
    events.push(ev("turn_end", Some("direct"), Some(final_text.clone())));
    emit(TurnProgress::TurnEnd {
        ended_with: "DIRECT".into(),
        final_text: final_text.clone(),
    });
    return Ok(TurnResult {
        final_text,
        events,
        ended_with: "DIRECT".into(),
    });
}
```

关键：非-pig 路径的 `payload` 是纯 `user_question`，**不追加 `pre_user_payload` 相位指令**。它就是一个普通的 LLM 调用，但经过 `self.api`（ProxyApiClient）→ `dispatch_in_process` → retry 逻辑。

### 3. PhasedRuntime 新增构造函数 `from_proxy_client`

**文件**: `crates/pigs-api/src/phased_runtime.rs`

新增一个接受外部 `ApiClient` 的构造函数，替代现有的 `from_resolved`（后者内部调用 `pigs_llm::create_client_for_endpoint`）：

```rust
/// 从外部 ApiClient 构建运行时（供 CLI 用，注入 ProxyApiClient）。
/// Build runtime from an external ApiClient (for CLI, injects ProxyApiClient).
pub fn from_client(
    api: Arc<dyn ApiClient>,
    remote_model: String,
    is_pig: bool,
    limits: RuntimeLimits,
    language: Language,
) -> Self {
    let wrapped_model = if is_pig {
        format!("{}-pig", remote_model)
    } else {
        remote_model.clone()
    };
    Self {
        api,
        remote_model,
        wrapped_model,
        tools: info_tool_registry(),
        limits,
        language,
        is_pig,
    }
}
```

`from_resolved` 和 `from_config` 保持不变（已有 `#[allow(dead_code)]`），但也会加上 `is_pig` 字段初始化（从 `resolved.is_pig` 读取）。

### 4. pigs-cli 依赖 pigs-proxy

**文件**: `crates/pigs-cli/Cargo.toml`

新增依赖：
```toml
pigs-proxy = { path = "../pigs-proxy" }
```

这不会引入循环依赖：`pigs-proxy` 依赖 `pigs-api`，`pigs-cli` 依赖 `pigs-proxy` + `pigs-api`，`pigs` 二进制依赖 `pigs-cli` + `pigs-proxy`。链路清晰：`pigs → pigs-cli → pigs-proxy → pigs-api → pigs-core`。

### 7. Agent::run_turn 统一走 PhasedRuntime

**文件**: `crates/pigs-cli/src/agent.rs`

**删除现有的 `is_pig` 分支**（lines 204-210），所有模型统一走 PhasedRuntime：

```rust
pub async fn run_turn(&mut self, user_input: &str) -> anyhow::Result<String> {
    self.run_turn_phased(user_input).await
}
```

**重构 `run_turn_phased`**（lines 698-794）：不再用 `PhasedRuntime::from_resolved`（内部建 pigs-llm 客户端），改为用 `PhasedRuntime::from_client` 注入 `self.api_client`：

```rust
async fn run_turn_phased(&mut self, user_input: &str) -> anyhow::Result<String> {
    let resolved = self.resolved_model()?;
    let limits = crate::phased_runtime::RuntimeLimits {
        max_tokens: resolved.max_tokens.unwrap_or(self.config.max_tokens),
        temperature: resolved.temperature.unwrap_or(self.config.temperature),
        ..Default::default()
    };

    // 用注入的 api_client 构建 PhasedRuntime（不再内部建 pigs-llm 客户端）
    // Build PhasedRuntime with the injected api_client
    let mut runtime = crate::phased_runtime::PhasedRuntime::from_client(
        Arc::clone(&self.api_client),
        resolved.remote_model.clone(),
        resolved.is_pig,
        limits,
        self.language,
    );

    // 注入工具注册表（与现有逻辑相同）
    let agent_tools = std::mem::replace(&mut self.tool_registry, pigs_core::ToolRegistry::new());
    runtime.tools = agent_tools;

    // ... 后续消息构建、progress sink、run_turn_with_progress 调用不变 ...
}
```

### 8. pigs 二进制注入 ProxyApiClient

**文件**: `crates/pigs/src/main.rs`

`run_api_and_cli` (line 226-278) 中，构建 `ProxyApiClient` 并注入 CLI：

```rust
async fn run_api_and_cli(args: &Args) -> anyhow::Result<()> {
    let config_path = /* 现有逻辑 */;
    let proxy_config = ProxyConfig::load(config_path.as_path())?;
    let proxy_config_arc = Arc::new(proxy_config.clone());

    // 构建 ProxyApiClient（供 CLI 和 HTTP 服务器共用）
    // Build ProxyApiClient (shared by CLI and HTTP server)
    let cli_client: Arc<dyn pigs_core::ApiClient> = Arc::new(
        pigs_proxy::proxy_client::ProxyApiClient::new(
            Arc::clone(&proxy_config_arc),
            pigs_proxy::protocol::Protocol::OpenAI,
            "pigs".to_string(),
        )
    );

    // 后台启动 pigs-proxy 服务器
    let runtime = pigs_proxy::build_phased_runtime(
        Arc::clone(&proxy_config_arc),
        "pigs",
        /* ... */
    )?;
    let proxy_config_clone = proxy_config.clone();
    let api_handle = tokio::spawn(async move {
        if let Err(e) = pigs_proxy::serve(proxy_config_clone, runtime).await { ... }
    });

    // 前台运行 pigs-cli，注入 ProxyApiClient
    let mut full_args = vec!["pigs".to_string()];
    full_args.extend(args.cli_args.iter().cloned());
    let code = pigs_cli::run_cli_with_client(full_args, Some(cli_client)).await;

    api_handle.abort();
    // ... 其余不变 ...
}
```

`--cli` 模式（line 152-169）和 one-shot 模式（line 178-190）也需要适配：它们目前直接调 `run_cli_from`，不经过 proxy。可以保持不变（注入 `None`，CLI 自建 pigs-llm 客户端），或者也注入 ProxyApiClient。**建议：`--cli` 和 one-shot 也注入 ProxyApiClient**，保持一致性。
