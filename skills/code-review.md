---
name: code-review
description: Review code changes for correctness, clarity, and safety
---

When the user asks for a code review:

1. Summarize the intent of the change first.
2. Check for correctness bugs, edge cases, and error handling gaps.
3. Flag unsafe operations, secret leakage, and permission issues.
4. Suggest concrete, minimal fixes with file references.
5. Prefer actionable comments over style nits unless style breaks readability.
