Chapter 07

错误处理

src/error.rs
统一错误类型 + ? 操作符。Rust 不靠异常，靠 Result<T, E>。

01 Result 与 ?

// 不用 ?
let content = match std::fs::read_to_string("file.txt") {
    Ok(c) => c,
    Err(e) => return Err(MiniAgentError::IoError(e.to_string())),
};

// 用 ?（需 From 实现）
let content = std::fs::read_to_string("file.txt")?;

02 MiniAgentError

┌──────────────────────────────────────────────────┐ │ MiniAgentError │ ├── LlmError LLM API 调用失败 │ ├── ToolError 工具执行失败 │ ├── NetworkError 网络请求失败 │ ├── ParseError JSON 解析失败 │ ├── IoError 文件 IO 失败 │ ├── MaxRoundsReached 达到最大轮次 │ └── ConfigError 配置错误 │ │ From<io::Error> / reqwest::Error / serde_json::Error │ type Result<T> = Result<T, MiniAgentError> └──────────────────────────────────────────────────┘

03 变体说明

变体场景
NetworkError底层：DNS、超时、TCP
LlmErrorAPI 层：4xx/5xx 或错误 JSON
MaxRoundsReached保护：循环达上限（含轮次数）
ToolError硬错误：工具不存在等
Tool 硬错误 vs 软错误？

硬错误：返回 Err（如找不到工具）。软错误：返回 Ok 但 is_error=true（如命令非零退出），错误信息进 tool 消息让 LLM 看到。

04 设计决策

为什么一个统一类型？

教学简化。pigs-core 拆成 ApiError + ToolError；本 crate 合并，一个 From 链就能 ? 到底。

为什么 thiserror？

#[error("...")] 自动实现 Display + Error，省掉几十行手写 boilerplate。

为什么 type Result<T>？

Rust 惯用：fn foo() -> Result<String> 即 Result<String, MiniAgentError>。

05 From 转换

impl From<std::io::Error> for MiniAgentError { /* IoError */ }
impl From<reqwest::Error> for MiniAgentError { /* NetworkError */ }
impl From<serde_json::Error> for MiniAgentError { /* ParseError */ }

CODE 源码拆解（src/error.rs）

1. thiserror 枚举

#[derive(Debug, Error)]
pub enum MiniAgentError {
    #[error("LLM 调用失败: {0}")]
    LlmError(String),
    // ...
    #[error("Agent 达到最大轮次限制 ({0} 轮)，强制停止")]
    MaxRoundsReached(u32),
}
  • #[error("...")]：自动 Display
  • {0}：元组变体字段
  • Debug + Error：可在 Result 里 ?

2. 七个变体对照调用点

变体 典型产生位置
LlmError call_with_retry HTTP 4xx / 业务失败
ToolError 缺参、未知工具、文件操作失败
NetworkError reqwest 连接失败耗尽重试
ParseError 响应 JSON / From serde_json
IoError From std::io::Error
MaxRoundsReached Agent::chat 打满轮次
ConfigError 缺 OPENAI_API_KEY

3. 三个 From

impl From<std::io::Error> for MiniAgentError { /* IoError */ }
impl From<reqwest::Error> for MiniAgentError { /* NetworkError */ }
impl From<serde_json::Error> for MiniAgentError { /* ParseError */ }

工具层常 故意 map_err 成更具体的 ToolError("读取文件失败...")，对模型更友好，而不是裸 IoError。

4. 类型别名

pub type Result<T> = std::result::Result<T, MiniAgentError>;

5. 硬错误 vs 软错误

类型 表现 例子
硬错误 Err(...) 冒泡 未知工具、超时、API 401
软错误 Ok 文本或 is_error=true bash 非零退出

Agent::chat 对 tools.execute 的 Err 会转成字符串 + is_error，不因单工具失败中止整次 chat；llm.chat 失败才会冒泡。
