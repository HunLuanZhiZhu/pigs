01 这是什么？

pigs-mini-agent 是一个自包含的、教学导向的通用 AI Agent crate。 它用不到 3000 行 Rust 代码展示了 AI Agent 最核心的工作原理——没有任何黑魔法， 不依赖任何其他 pigs-* crate。

如果你想知道 Claude Code、Cursor、Codex 这些工具「底下到底在干什么」， 读这个 crate 就够了。

02 核心循环

Agent 的全部「智能」来自一个循环。代码本身不做任何决策——所有思考来自 LLM，Agent 只是 LLM 的手脚。

1
用户发消息
加入对话历史
2
调用 LLM
把历史 + 工具列表发给大模型
3
LLM 回复
判断是否包含工具调用
├── 纯文本 → 返回给用户 ✓
└── 工具调用 → 执行 → 结果入历史 → 回到 2
✓
结束或达上限
循环直到不再需要工具，或 max_rounds

04 快速开始

# 设置 API 密钥（任选一个供应商）
export OPENAI_API_KEY="sk-xxxx"

# 运行示例
cargo run --example chat
use pigs_mini_agent::{Agent, LlmClient, create_default_tools};

#[tokio::main]
async fn main() -> pigs_mini_agent::Result<()> {
    let llm = LlmClient::from_env()?;
    let tools = create_default_tools();
    let mut agent = Agent::new(llm, tools);
    let response = agent.chat("帮我创建一个 hello.txt 文件").await?;
    println!("{response}");
    Ok(())
}

05 关键设计决策

为什么自包含？

不依赖 pigs-core / pigs-llm 等。读者只看这一个 crate 就能理解完整 Agent 循环，不需要在多个 crate 之间跳转。

为什么用 OpenAI 兼容格式？

OpenAI、DeepSeek、Qwen、Kimi、Ollama 都实现了这个格式。一个客户端就能对接所有供应商。

为什么用非流式？

流式响应需要处理 SSE 和工具参数碎片重组，增加约 200 行复杂代码。教学版优先展示核心逻辑。

为什么 edit_file 用唯一匹配而非行号？

「行号是陷阱，模型数错一行就悄悄改错地方。用唯一片段锚定，失败可恢复、成功可验证。」—— CoreCoder
